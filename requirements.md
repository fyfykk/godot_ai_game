# 游戏功能与需求文档 v0.5（2026-03-17）

## 概述
- 目标：汇总当前已实现功能与需求边界，形成可验收的玩法清单与参数源。
- 核心循环：出生 → 探索/战斗 → 交互（门/宝箱/梯子）→ 收集 → 到达出口 → 结算与存档。
- 平台与画面：PC，Godot 4.6；1280×720（16:9），像素风低分辨率材质。

## 当前功能清单
- 关卡流程与结算：主菜单进入关卡；胜利/失败弹窗；胜利结算收藏品入库；失败随机保留 1 个。
- 角色移动与镜头：A/D 左右移动；爬梯（W/S）上下；跳跃禁用；镜头跟随与边界限制。
- 交互系统：最近可交互目标锁定；同距取左；E 触发；交互提示仅显示在目标上。
- 互动提示组件：统一提示样式、背景、描边与偏移参数。
- 门系统：R 开关门；开门可修复（长按 E 进度环）；关门可被敌人攻击；门血条位置同步；门自带墙体表现。
- 特殊门与解谜：特殊门需解锁；电路连线小游戏；每局每扇门谜题随机且本局固定；可关闭不解锁。
- 梯子系统：随机生成；顶层梯子初始锁定；长按 E 按秒消耗金币解锁进度；解锁后可进入上层。
- 宝箱系统：普通宝箱长按 E 开启；单箱最多开启 3 次；掉落计划由 GameRoot 统一生成。
- 密码宝箱：输入 4 位数字密码；错误提示；成功后掉落升级或收藏品。
- 出口：长按 E 触发撤离；进入后置交互期（倒计时）结束结算。
- 战斗系统：自动攻击模块（子弹/近战/范围魔法）；攻击参数可升级；可解锁新攻击模块。
- 敌人与 Boss：敌人追击与攻击；可爬梯追踪；Boss 到达最大时间后生成并强化。
- 刷怪逻辑：按时间曲线与上限生成；在平台有效范围内落点；后置期刷怪节奏调整。
- 掉落与经济：金币与收藏品掉落；敌人低血时概率掉血包；金币与收藏品计数显示。
- 收藏品与成长：本局背包累积；胜利入库；失败随机保留；红色收藏品解锁开局攻击。
- UI 与 HUD：HP/敌人数、金币/收藏品数、血月/撤离进度、升级/收藏品三选一、暂停/图鉴。
- 小地图：按镜头视野逐步发现宝箱/门/梯子/墙；显示边界墙与出口位置。
- 地图编辑器：编辑平台/梯子/门/特殊门/出生/出口/宝箱；自定义地图可删除并自动顺延编号。
- 字体系统：全局 UI 字体统一为开源中文字体（Web 兼容）；新建控件自动覆盖字体。
- 工具与发布：配置打包脚本；Windows/Web 导出；局域网 HTTP/HTTPS 访问工具与自签证书。

## 交互与输入规则
- 交互选择：仅最近可交互物体可触发与显示提示；同距时选择更靠左的物体。
- 门提示逻辑：开门时，靠近开门位置只显示 E；靠近门本体只显示 R；关门时只显示 R。
- 交互取消：移动/离开或条件失效会中断交互进度。
- 解谜输入：解谜界面打开时，玩家移动与交互输入被锁定；ESC 不可打开暂停菜单。
- 数字密码：键盘数字/屏幕按钮输入；退格删除；Enter 提交。

## 需求与验收标准
- 门修复与开关
  - 门最大血量 300；每次修复 +100；开门不可被攻击，关门可被攻击并可被打开。
  - E 修复仅在门开且血量未满时显示；R 仅在门本体附近显示。
  - Boss 与普通敌人均能攻击关门。
- 特殊门解锁
  - 触发解谜后生成电路连线小游戏；完成即解锁门。
  - 每局随机种子，单门在本局内谜题固定；下一局重新随机。
  - 解谜可手动关闭，但不解锁门。
- 梯子解锁
  - 顶层梯子初始锁定；长按 E 按秒消耗金币解锁进度；解锁后可进入上层。
  - 解锁中移动/金币不足立即中断；进度环与数值一致。
- 宝箱与掉落
  - 宝箱单个最多开启 3 次；每次开箱掉落计划由 GameRoot 统一生成。
  - 掉落物贴地落点，拾取自动生效并移除。
- 密码宝箱
  - 输入 4 位密码后判定；错误提示；正确则发放升级或收藏品。
- 敌人与 Boss
  - 敌人追击玩家并在范围内攻击；无法触达时使用梯子追踪。
  - Boss 到达最大时间后生成；血量/伤害/速度来自角色配置表并随时间缩放。
- 交互唯一性
  - 玩家同时只可交互一个目标；提示也仅显示在该目标。
- 关卡生成
  - 多层平台与梯子连接；左右边界墙贯通，从最底层顶面延伸到最高层再向上 1 层。
  - 房间分隔墙与门成对生成；宝箱避让梯子。
- UI 与暂停
  - 三选一界面打开时暂停按钮不可用；1/2/3 快捷选择有效。
  - 暂停菜单可进入收藏品图鉴并返回。
- 小地图探索
  - 仅记录镜头范围内出现过的宝箱/门/梯子/墙；离开视野后仍保留显示。
- 地图编辑器
  - 支持新建与编辑自定义地图；仅自定义地图可删除。
  - 自定义列表命名为“自定义1/2/3”，删除后编号自动顺延。
- 全局字体
  - 所有 UI 文本使用统一中文字体；运行时动态创建控件自动套用字体。

## 验收用例（示例）
- 交互唯一性：玩家站在门与宝箱之间，提示仅显示最近者；同距时显示左侧。
- 门提示：门开且玩家靠近开门位置只显示 E；靠近门本体只显示 R。
- Boss 攻门：Boss 与关门接触时可触发门受击与特效。
- 梯子解锁：金币不足/移动会中断解锁；解锁后顶层可进入。
- 三选一：面板打开时暂停按钮隐藏且不可点击；1/2/3 可直接选中。
- 解谜锁输入：解谜界面打开时玩家无法移动或交互，ESC 不可打开暂停菜单。

## KPI（观察与目标）
- 首关完成率≥65%，第3关≥40%，第5关≥25%
- 单局时长 5–8 分钟；每分钟击杀 2–6
- 首局货币产出约为基础升级成本的 20–30%
- 交互成功率（门修复）≥85%
- 主循环帧时长≤16.7ms（60FPS）

## 事件与埋点（建议）
- 交互：door_repair_start/complete、door_toggle、ladder_unlock_progress
- 升级：upgrade_panel_open/choice_made、unlock_attack_selected
- 探索：minimap_discover_{chest|door|ladder|wall}
- 掉落：medkit_drop/show/pick、pickup_spawn/pick
- 解谜：puzzle_open/close/complete、password_submit_success/fail
