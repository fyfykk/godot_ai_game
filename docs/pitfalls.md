# 易错点

## 背包拖拽旋转时矩形变正方形

### 现象
- 2×3 物品在拖拽按 R 旋转时临时显示为 3×3，松手后恢复正常

### 原因
- 背包物品节点或其子节点设置了 custom_minimum_size
- 在拖拽旋转过程中，布局系统使用最小尺寸约束了节点宽高，导致宽度无法缩小而被强制为正方形

### 解决
- 旋转拖拽路径中不要设置 custom_minimum_size
- 仅更新 size，并在需要时使用 set_deferred("size", ...) 确保尺寸刷新

### 定位方法
- 打印旋转时的 sz、目标尺寸与实际 node.size
- 若 sz 为 2×3 但 node.size 为 3×3，优先排查 custom_minimum_size 与容器布局约束
